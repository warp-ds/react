import { Tab, Tabs, TabPanel } from '../src';

# Tabs

```jsx example
<Tabs onChange={(e) => console.log(e)}>
    <Tab label="Tab 1" name="one" isActive />
    <Tab label="Tab 2" name="two" />
    <Tab label="Tab 3" name="three" />
</Tabs>
<TabPanel name="one">Panel 1 content</TabPanel>
<TabPanel name="two">Panel 2 content</TabPanel>
<TabPanel name="three">Panel 3 content</TabPanel>
```

## Import

```js
import { Tab, Tabs, TabPanel } from '@warp-ds/react';
```

## Example

The following example demonstrates how the Tab, Tabs, and TabPanel components
can be used to switch between panels.

```jsx example
function Example() {
  return (
    <div>
      <Tabs active="four">
        <Tab label="Tab 1" name="four" />
        <Tab label="Tab 2" name="five" />
        <Tab label="Tab 3" name="six" />
      </Tabs>
      <div className="mb-16">
        <TabPanel name="four">Panel 1 content</TabPanel>
        <TabPanel name="five">Panel 2 content</TabPanel>
        <TabPanel name="six">Panel 3 content</TabPanel>
      </div>
    </div>
  );
}
```

## Technical details

You can set the active tab in one of two ways:

1. By passing the name of the active Tab to the Tabs component using the active
   attribute.
2. By setting isActive on the active Tab.

The first Tab is active by default if you don't specify an active Tab.

It is important that all children of Tabs are Tab components.

Note that name attributes will be used to generate id attributes (prefixed with
warp-tab- and warp-tabpanel-), and therefore they must be unique throughout
the entire DOM. This is because aria-controls and aria-labelledby rely on id
attributes, and they are required for the tabs to be ARIA compliant.

## Accessibility

Focus management and ARIA attributes are handled automatically.

## Visual options

There are two visual variations, default and contained. In either case, an icon
can be provided for each tab by passing it in as a child to the component. By
default, provided icons will be placed to the left of the label. You may move
the icons to above by using the `over` prop.

### default

```jsx example
<Tabs>
  <Tab label="Tab 1" name="one" isActive />
  <Tab label="Tab 2" name="two" />
  <Tab label="Tab 3" name="three" />
</Tabs>
```

### default with left icons

```jsx example
function Example() {
  return (
    <Tabs>
      <Tab label="Tab 1" name="one" isActive>
        <IconPlus16 />
      </Tab>
      <Tab label="Tab 2" name="two">
        <IconPlus16 />
      </Tab>
      <Tab label="Tab 3" name="three">
        <IconPlus16 />
      </Tab>
    </Tabs>
  );
}
```

### default with icons over the label

```jsx example
function Example() {
  return (
    <Tabs>
      <Tab label="Tab 1" name="one" over isActive>
        <IconPlus16 />
      </Tab>
      <Tab label="Tab 2" name="two" over>
        <IconPlus16 />
      </Tab>
      <Tab label="Tab 3" name="three" over>
        <IconPlus16 />
      </Tab>
    </Tabs>
  );
}
```
